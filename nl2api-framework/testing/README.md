# Testing & Evaluation 🧪

This directory contains all the scripts and components used to evaluate the performance of the trained models, as well as to benchmark against external models such as LLaMA.

---

## 📂 Structure Overview

The testing pipeline is divided into three main parts:

* **Model Evaluation (Local)**

  * `test_tool_caller.py`
  * `test_mlp.py`

* **LLaMA Evaluation (External / Baseline)**

  * `main_client_presets.py`
  * `main_server_presets.py`
  * `llm_adapter.py`
  * `preset_tools.py`

* **Results Processing**

  * `evaluate_LLaMA.py`

---

## 🧠 Local Model Evaluation

### 1. Tool Calling Evaluation

**File:** `test_tool_caller.py`

Evaluates the fine-tuned model (LoRA-based) on the tool calling task.

### What it does:

* Loads the trained model
* Runs inference on a test dataset
* Compares predicted tool calls against ground truth
* Logs qualitative and/or structured outputs

### Output:

* Numerical results alongside raw predictions

---

### 2. MLP Evaluation

**File:** `test_mlp.py`

Evaluates the classifier-based approach (MLP head).

### What it does:

* Loads the trained classifier
* Predicts tool usage from queries
* Compares predictions with labels

### Output:

* Predictions
* Accuracy or related metrics (depending on configuration)

### IMPORTANT
For this test script, the `.pt` file containing the trained MLP weights is required. Make sure the model checkpoint is available and that the path is correctly specified in the script before running the evaluation.

---

## 🤖 LLaMA Evaluation (Baseline)

This section provides a way to evaluate a large model (e.g., LLaMA 3.3) as a baseline for comparison. For this, we create a MCP server approach, where LLaMA 3.3 selects the tool from a tool pool specified in a MCP server.

Take in mind that an Ollama server was used in our experiments, and a Ollama configuration file is expected by these scripts.

### Components

#### `main_client_presets.py`

* Acts as the client interface
* Sends queries to the model
* Handles request formatting

#### `main_server_presets.py`

* Wraps the MCP server

#### `llm_adapter.py`

* Abstraction layer between the evaluation pipeline and the model
* Allows switching models without changing the evaluation logic
* Handles prompt formatting and response parsing

#### `preset_tools.py`

* Defines the tools available in the MCP server.
* Must be consistent with:

  * Dataset generation
  * Evaluation expectations

---

## 📊 Results Evaluation

### `evaluate_llama.py`

This script processes the outputs generated by the LLaMA evaluation pipeline.

### What it does:

* Reads model predictions
* Compares them against ground truth
* Computes numerical evaluation metrics

### Metrics may include:

* Accuracy
* Tool selection correctness
* Argument correctness

---

## ⚠️ Important Considerations

### Consistency is Critical

Ensure that the following elements are aligned:

* `tools_spec.json`
* Test dataset
* Model outputs format

Any mismatch will lead to incorrect evaluation.

---

### File Configuration

Some scripts may require manual adjustments:

* Input/output file paths
* Model endpoints (for LLaMA server/client setup)
* Dataset selection

---

## 🔁 Typical Workflow

### 1. Evaluate Local Models

* Tool calling:

  ```bash
  python test_tool_caller.py
  ```

* MLP:

  ```bash
  python test_mlp.py
  ```

---

### 2. Evaluate LLaMA Baseline

1. Run the client:

   ```bash
   python main_client_presets.py
   ```

2. Compute metrics:

   ```bash
   python evaluate_llama.py
   ```

---

## 🧾 Summary

* `test_tool_caller.py` → Evaluates LoRA-based tool calling
* `test_mlp.py` → Evaluates classifier-based approach
* `main_* + adapter + tools_spec` → LLaMA evaluation pipeline
* `evaluate_llama.py` → Computes final numerical results

This setup allows direct comparison between:

* Fine-tuned small models (Gemma 2)
* Large general-purpose models (LLaMA)
